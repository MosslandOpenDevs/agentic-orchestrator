### 2. Technical Architecture

*   **Frontend:** React.js – Provides a responsive, interactive interface for users to interact with the system and view risk scores.  Its component-based architecture aligns well with the modular design of our solution.
*   **Backend:** Python (with FastAPI) – Offers rapid development, scalability, and a robust ecosystem for AI integration and blockchain interactions.
*   **Database:** PostgreSQL – Provides strong data integrity, ACID compliance, and efficient querying capabilities – crucial for managing complex transaction data and risk scores.
*   **Blockchain Integration:** Ethereum (primarily) with Polygon integration for lower transaction fees and faster processing during initial verification. We'll leverage Web3.js for smart contract interactions.
*   **External APIs:**
    *   Chainlink for oracle data feeds (price feeds, market data).
    *   Etherscan for transaction monitoring and blockchain analysis.
    *   OpenAI API (GPT-5) for advanced pattern recognition and anomaly detection.
*   **System Architecture Diagram:** (Text Description) – A layered architecture consisting of a User Interface (React), a Backend API (Python/FastAPI), a Data Processing Engine (Python/AI Libraries), a Blockchain Interaction Layer (Web3.js), and a Database (PostgreSQL).  Data flows from the UI to the API, which then interacts with the AI engine, blockchain, and database.
