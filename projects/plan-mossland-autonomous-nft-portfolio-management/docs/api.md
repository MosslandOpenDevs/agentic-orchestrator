# NFT Portfolio Analysis API Documentation

## 1. Overview

This API provides endpoints for retrieving portfolio data, analyzing portfolio positions, and obtaining asset price data. It leverages GPT-5 recommendations for portfolio analysis and offers a robust interface for managing and understanding your NFT portfolio.

## 2. Authentication Details

All API requests require an API key passed in the `X-API-Key` header.  You can obtain your API key from the [API Key Management Portal](https://example.com/apikey).  Ensure your key is kept confidential.

## 3. Base URL Configuration

The base URL for all API requests is:

`https://api.example.com/v1`

## 4. Endpoints

### 4.1. GET /api/nftHolders

*   **Method:** `GET`
*   **Path:** `/api/nftHolders`
*   **Description:** Retrieves a list of all NFT holders.
*   **Request Parameters/Body:** None
*   **Response Format:** JSON Array
    *   Each object in the array represents an NFT holder and contains the following fields:
        *   `address`: (String) The Ethereum address of the NFT holder.
        *   `nftCount`: (Integer) The number of NFTs held by the address.
*   **Example Request:**
    ```
    GET /api/nftHolders
    X-API-Key: YOUR_API_KEY
    ```
*   **Example Response:**
    ```json
    [
      {
        "address": "0x1234567890abcdef",
        "nftCount": 15
      },
      {
        "address": "0x9876543210fedcba",
        "nftCount": 3
      }
    ]
    ```

### 4.2. GET /api/portfolioPositions

*   **Method:** `GET`
*   **Path:** `/api/portfolioPositions`
*   **Description:** Retrieves a list of all portfolio positions.
*   **Request Parameters/Body:** None
*   **Response Format:** JSON Array
    *   Each object in the array represents a portfolio position and contains the following fields:
        *   `assetAddress`: (String) The Ethereum address of the asset.
        *   `quantity`: (Integer) The quantity of the asset held.
        *   `currentPrice`: (Float) The current price of the asset.
        *   `totalValue`: (Float) The total value of the position.
*   **Example Request:**
    ```
    GET /api/portfolioPositions
    X-API-Key: YOUR_API_KEY
    ```
*   **Example Response:**
    ```json
    [
      {
        "assetAddress": "0xabc1234567890abcdef",
        "quantity": 100,
        "currentPrice": 150.75,
        "totalValue": 15075.00
      },
      {
        "assetAddress": "0xdef9876543210abcdef",
        "quantity": 50,
        "currentPrice": 225.50,
        "totalValue": 11275.00
      }
    ]
    ```

### 4.3. GET /api/assets/{assetAddress}

*   **Method:** `GET`
*   **Path:** `/api/assets/{assetAddress}`
*   **Description:** Retrieves price data for a given asset address.
*   **Request Parameters/Body:** None
*   **Response Format:** JSON
    *   `currentPrice`: (Float) The current price of the asset.
    *   `lastUpdated`: (String) Timestamp of the last price update (ISO 8601 format).
*   **Example Request:**
    ```
    GET /api/assets/0xabc1234567890abcdef
    X-API-Key: YOUR_API_KEY
    ```
*   **Example Response:**
    ```json
    {
      "currentPrice": 165.20,
      "lastUpdated": "2023-10-27T10:30:00Z"
    }
    ```

### 4.4. POST /api/analyzePortfolio

*   **Method:** `POST`
*   **Path:** `/api/analyzePortfolio`
*   **Description:** Analyzes the portfolio based on risk parameters and GPT-5 recommendations.
*   **Request Parameters/Body:** JSON
    *   `riskTolerance`: (String)  "low", "medium", or "high".  Determines the risk level for the analysis.
    *   `investmentHorizon`: (Integer) The investment horizon in days.
    *   `assetWhitelist`: (Array of Strings)  A list of asset addresses to include in the analysis.
    *   `assetBlacklist`: (Array of Strings) A list of asset addresses to exclude from the analysis.
*   **Response Format:** JSON
    *   `recommendations`: (Array of Strings)  A list of recommendations generated by GPT-5.
    *   `riskScore`: (Integer) A calculated risk score based on the portfolio.
*   **Example Request:**
    ```
    POST /api/analyzePortfolio
    X-API-Key: YOUR_API_KEY
    Content-Type: application/json

    {
      "riskTolerance": "medium",
      "investmentHorizon": 365,
      "assetWhitelist": ["0xabc1234567890abcdef", "0xdef9876543210abcdef"],
      "assetBlacklist": ["0x1122334455667788"]
    }
    ```
*   **Example Response:**
    ```json
    {
      "recommendations": [
        "Consider diversifying into low-risk assets.",
        "Rebalance your portfolio to align with your medium risk tolerance."
      ],
      "riskScore": 65
    }
    ```

## 5. Error Codes and Handling

| Code    | Description                               | Action                               |
| :------ | :--------------------------------------- | :---------------------------------- |
| 400      | Bad Request - Invalid input parameters   | Validate request body and parameters |
| 401      | Unauthorized - Invalid API key          | Provide a valid API key             |
| 404      | Not Found - Resource not found           | Check asset address or endpoint path |
| 429      | Too Many Requests - Rate limit exceeded | Implement retry logic or back off    |
| 500      | Internal Server Error                    | Contact API support                  |

## 6. Rate Limiting Info

The API is rate-limited to 100 requests per minute per API key.  This is to prevent abuse and ensure fair usage.  If you exceed the rate limit, you will receive a 429 error.  You can adjust your rate limits in the [API Key Management Portal](https://example.com/apikey).
